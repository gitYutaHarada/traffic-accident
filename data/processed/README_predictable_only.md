# honhyo_clean_predictable_only.csv

## 概要
予測モデル用に最適化されたデータセット。予測時に取得不可能な当事者B（相手側）の属性情報を削除し、**予測可能な特徴量のみ**で構成されています。

## データ仕様
- **データ行数**: 1,895,275行
- **カラム数**: 33列
- **目的変数**: `死者数`（バイナリ: 0 or 1）
- **ベースデータ**: `honhyo_clean_binary.csv`

## 削除されたカラム（3列）

以下のカラムは**予測時に取得不可能**なため削除されました：

| カラム名 | 削除理由 |
|---------|---------|
| `年齢（当事者B）` | 相手の年齢は事故前に予測不可能 |
| `当事者種別（当事者B）` | 相手の車種（乗用車/トラック/二輪車等）は事故前に予測不可能 |
| `用途別（当事者B）` | 相手の用途（事業用/自家用）は事故前に予測不可能 |

## 保持された当事者B関連カラム（3列）

以下のカラムは**道路構造として解釈可能**なため保持されました：

| カラム名 | 保持理由 |
|---------|---------|
| `一時停止規制　標識（当事者B）` | 相手側の道路に設置された標識＝道路構造の一部 |
| `一時停止規制　表示（当事者B）` | 相手側の道路の路面表示＝道路構造の一部 |
| `速度規制（指定のみ）（当事者B）` | 相手側の道路の速度規制＝道路構造の一部 |

これらは地図データなどから事前に把握可能な情報です。

## 全33カラムの構成

### 1. 目的変数（1列）
- `死者数` - 予測対象（0: 負傷のみ, 1: 死亡）

### 2. 当事者A関連（6列）
- `年齢（当事者A）` - 自分の年齢層
- `当事者種別（当事者A）` - 自分の車種
- `用途別（当事者A）` - 自分の用途
- `一時停止規制　標識（当事者A）`
- `一時停止規制　表示（当事者A）`
- `速度規制（指定のみ）（当事者A）`

### 3. 当事者B関連（3列）※道路構造として保持
- `一時停止規制　標識（当事者B）`
- `一時停止規制　表示（当事者B）`
- `速度規制（指定のみ）（当事者B）`

### 4. 環境・時間条件（9列）
- `天候`, `昼夜`, `曜日(発生年月日)`, `祝日(発生年月日)`
- `地形`, `地点　経度（東経）`, `地点　緯度（北緯）`
- `地点コード`, `市区町村コード`

### 5. 道路構造（8列）
- `路線コード`, `路面状態`, `車道幅員`
- `道路形状`, `道路線形`, `歩車道区分`
- `中央分離帯施設等`, `ゾーン規制`

### 6. 事故状況（6列）
- `事故類型`, `衝突地点`, `信号機`
- `都道府県コード`, `警察署等コード`, `発生日時`

## 使用目的

このデータセットは以下の用途に最適です：

1. **予測モデルの学習**: 予測時に利用可能な特徴量のみで学習
2. **リアルタイム予測**: 運転中に取得可能な情報のみを使用
3. **データリーク防止**: 事故後にしか分からない情報を完全除外
