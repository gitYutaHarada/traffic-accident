# 実験レポート: 事故類型（Leakage）削除後のモデル性能評価

## 実験概要
- **目的**: 事故発生後にしか分からない情報（`事故類型`）を削除し、事故発生前に予測可能な真の精度を計測する。
- **データ**: `honhyo_clean_no_leakage.csv` (事故類型なし)
- **特徴量**: エリアID(n=50) + `interaction_type_speed` (車種A×速度A)

## 📊 実験結果

| モデル | AUC | Recall | 評価 |
| :--- | :--- | :--- | :--- |
| Leakageあり (参考) | 0.8878 | 0.7381 | ※過大評価の可能性 |
| **No Leakage (今回)** | **0.8739** | **0.7416** | **実用的かつ健全** |

### 考察
1. **精度の維持**: 強力なLeakage変数（事故類型）を削除したにもかかわらず、AUCの低下はわずか **-0.014** に留まりました。
2. **Recallの向上**: 驚くべきことに、Recall（死亡事故検知率）は **0.7381 -> 0.7416** へと向上しました。
    - **理由**: 「事故類型（人対車など）」という安易な答えがなくなったことで、モデルが「道路環境（速度規制や一時停止）」や「車種特性」といった**本質的な危険因子**をより真剣に学習するようになったためと推測されます。
3. **重要因子の変化**: Feature Importanceのトップが「速度規制（当事者B）」や「路線コード」になりました。これは「どのような環境で事故が起きるか」を捉えている証拠です。

現在のモデルは、事故発生前の情報のみを使用しており、かつ高い検出能力（Recall 74%）を持っています。**事故予測・予防システムとして十分に信頼できるモデル**となりました。
